Timecode Notes

v0.7.0-alpha

What this is

A lightweight note-taking tool for post-production review sessions. Start the clock when your pass starts, type freely, and every note is stamped with the timecode you began writing it.

Exports

  • .txt, .rtf (for reviewers), .json, a Markerbox-compatible CSV for Premiere, DaVinci Resolve EDL, a generic EDL, and a best-effort FCPXML

Storage

Everything lives in this browser's local storage on this device. Use Preferences to back up or move your data to another machine.

This app was built with Claude. This note will be removed once the team finishes reviewing and refining the code.

While recording

Keep typing — tags never take focus away from your note. Hover any earlier note to re-tag it, or click its text to fix a typo, without breaking stride.

Long notes

Press ⌘/Ctrl+Enter, or Tab, instead of Enter to log a note that spans from when you started typing to that moment — useful for a segment rather than a single frame.

Minimal mode

Turn this on in Settings when you want the screen stripped down to just the timecode and the note you're writing.

Projects

Give a project a fixed frame rate, start timecode, and metadata if every pass in it shares the same spec — new sessions there pick it up automatically.

Timecode drifted?

Use "Timecode Adjust" on the review screen to shift every note in a session at once if the clock started a beat early or late.

2POP sync

Turn this on in Settings before starting a session to skip manually timing the start against picture. The app listens for a 1kHz reference tone, starts the clock exactly 2 seconds before your session's start TC the instant it's heard, then asks you to confirm before you can start typing notes — watch the clock count up to your start TC as a live check that it synced correctly. This needs microphone access, which browsers only grant on a secure page (https://, or http://localhost) — opening the downloaded file directly won't prompt for it at all.

Exports for NLEs

DaVinci Resolve's export has been tested against a real install and works, colors included. Premiere Pro cannot import marker data on its own at all — not a gap in this app, a real limitation confirmed by Adobe's own community and Premiere's plugin SDK. The free Markerbox extension solves this by running inside Premiere; export "Premiere (via Markerbox)" and import that CSV through it. Final Cut Pro reads FCPXML directly. The "Generic EDL" is the classic locator convention some Avid and general-purpose tools expect.

Sharing notes with reviewers

The .rtf export is built for people, not NLEs — a clean formatted document that opens directly in Word, Pages, LibreOffice, or Google Docs, no import step needed.
